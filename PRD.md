# PRD — summary4u Mac 桌面 App

> 版本: v0.1 (待守一审核)
> 创建日期: 2026-06-07
> 范围: 把现有 WebUI 封装为 Mac 桌面 App,新增系统级集成

---

## 1. 背景与目标

### 1.1 当前形态的问题
summary4u 是一个 Python 命令行 + WebUI(FastAPI + localhost:8000) 的音视频总结工具。
对**自用**和**高频使用**场景,现有形态有几个明显摩擦:

| 摩擦点 | 痛苦程度 | 频次 |
|---|---|---|
| 每次要开终端跑 `start_webui.sh` | 高 | 每天 |
| 浏览器标签页淹没在众多 tab 里,要找半天 | 高 | 每天 |
| 想边看视频边记笔记时,得切换窗口粘贴 URL | 中 | 每天 |
| 在 Finder 看到 mp3 文件想转录,得拖进网页上传区 | 中 | 每周数次 |
| 关闭浏览器窗口服务就停了,还得手动重启 | 低 | 偶尔 |

### 1.2 目标
**把 summary4u 变成一个"住在菜单栏、随时按快捷键就能用"的 Mac 桌面工具**。
让用户在 5 秒内开始一次转录任务,不离开当前工作流。

### 1.3 非目标 (Non-Goals)
- ❌ 跨平台(Mac 优先,Win/Linux 暂不考虑)
- ❌ 公开分发(暂只自用,不签名/不上 Mac App Store)
- ❌ 重新设计后端逻辑(底层 Whisper/AI 总结能力复用,只做壳)
- ❌ 多用户/账号系统(单用户本地使用)
- ❌ 实时协作/云同步

## 3. 核心功能 (MVP 范围)

### 3.1 必须有 (Must-Have) — V1.0

**F-1 菜单栏常驻 (Menu Bar)**
- App 启动后菜单栏出现图标
- 状态指示:无任务=灰色圆点 / 有任务运行=绿色脉冲 / 全部完成=绿色静态
- 点击菜单栏图标 → 弹出小型 popover 窗口
- 菜单项:打开主窗口 / 最近 3 个任务 / 退出

**F-2 全局快捷键 (Global Hotkey)**
- 默认 `⌘⇧S`(可在偏好设置改)
- 任意应用下按下 → 弹出"快速输入"小浮窗(类似 Spotlight 风格)
- 浮窗内:URL/文件路径输入框 + 模板下拉 + 提交按钮
- 提交后浮窗自动关闭,后台开始任务

**F-3 Finder 拖放 (Drop Target)**
- App 接受两种拖放:
  - 拖到 Dock 图标 → 自动转录文件
  - 拖到主界面的拖放区 → 转录文件
- 支持多文件批量拖入
- 支持拖入 `.txt / .md` 文件(作为已转录文本直接总结,跳过 ASR 步骤)

**F-4 主界面 (Main Window)**
布局自上而下:
```
┌──────────────────────────────────────┐
│  [logo] summary4u          [⚙设置]    │  ← Header,56px
├──────────────────────────────────────┤
│  ┌──────────────────────────────┐   │
│  │ 输入 URL 或文件路径...         │   │  ← URL/Path 输入框
│  │ [模板▼] [✨ 总结]              │   │
│  └──────────────────────────────┘   │
│                                      │
│  ┌──────────────────────────────┐   │
│  │                              │   │
│  │      📁 拖放文件到这里        │   │  ← 大拖放区(支持批量)
│  │      或点击选择文件          │   │
│  │                              │   │
│  └──────────────────────────────┘   │
│                                      │
│  当前任务                              │
│  ┌──────────────────────────────┐   │
│  │ 🎙️  视频标题 [████████░░] 78% │   │  ← 实时进度
│  └──────────────────────────────┘   │
│                                      │
│  历史记录                  [查看全部]  │
│  • 视频A_总结.md · 2小时前   [打开]   │
│  • 音频B_总结.md · 昨天      [打开]   │
└──────────────────────────────────────┘
```

**F-5 任务进度 + 系统通知**
- 主界面显示当前运行任务的实时进度条
- 任务完成 → 弹系统通知(标题:"summary4u · 任务完成",内容含文件路径)
- 通知点击 → 打开主窗口并定位到该结果

**F-6 偏好设置 (Preferences)**
- 全局快捷键自定义
- 默认 Whisper 模型(tiny/base/small/medium)
- 默认总结模板
- 输出目录
- API Key 管理(复用现有 config.json)

**F-7 启动与生命周期**
- 开机自启动(可在偏好设置开关)
- 启动后菜单栏立即可见,主窗口默认隐藏(按需打开)
- 关闭主窗口 ≠ 退出 App(后台继续监听快捷键/拖放)

### 3.2 锦上添花 (Nice-to-Have) — V1.1+

- 剪贴板自动监听(复制 URL 弹"是否转录"提示)
- 菜单栏 popover 内直接查看最近 3 条任务状态
- 多语言界面(中英切换)
- Dark Mode 适配(沿用 DESIGN.md 的色板)
- 任务队列(同时处理多个任务)

### 3.3 显式不做 (Out of Scope)
- ❌ Windows / Linux 适配
- ❌ 公证/Apple Developer 签名(只自用,右键打开)
- ❌ 任何云端同步/账号体系
- ❌ 替换现有 WebUI 的后端实现(只是套壳)
- ❌ 重新设计 prompts 模板(沿用现有 7 套)

## 5. 技术方案概要

### 5.1 技术栈
- **壳**: PyWebView 4.x(Python 调用 macOS WKWebView)
- **打包**: PyInstaller(产出 .app,5-8MB)
- **菜单栏图标**: rumps(Python 库,专门做 macOS 菜单栏 App)
- **全局快捷键**: pynput 监听 + 备用 rumps 自带
- **Dock 拖放**: PyWebView 的 `webview.start(debug=...)` 配合 HTML5 drag-and-drop API
- **系统通知**: pync(Mac 原生通知)
- **后端**: 现有 FastAPI + src/ 模块**零修改**
- **前端**: 复用现有 `templates/` + `static/`,只做 3 处小改:
  1. 主窗口布局改为 PRD §3.1 F-4 描述的结构
  2. 进度条改用 EventSource 实时推送(用现有 polling 兜底)
  3. 适配 WKWebView 的拖放 API

### 5.2 架构图
```
┌──────────────────────────────┐
│   PyWebView 主壳 (Mac)        │
│  ┌────────────────────────┐  │
│  │  WKWebView → 复用      │  │
│  │  现有 WebUI 静态资源    │  │
│  └────────────────────────┘  │
│  ┌──────────┐ ┌───────────┐  │
│  │ rumps    │ │ pynput    │  │ ← 菜单栏 + 全局快捷键
│  │ (菜单栏) │ │ (快捷键)  │  │
│  └──────────┘ └───────────┘  │
│  ┌──────────┐ ┌───────────┐  │
│  │ pync     │ │ Dock 拖放 │  │ ← 通知 + Finder 拖放
│  │ (通知)   │ │           │  │
│  └──────────┘ └───────────┘  │
│  ┌────────────────────────┐  │
│  │  FastAPI 后端          │  │ ← 现有代码,零修改
│  │  (uvicorn 启动)         │  │
│  └────────────────────────┘  │
└──────────────────────────────┘
```

### 5.3 启动流程
1. PyInstaller 启动 → 启动 `main_app.py`
2. `main_app.py` 启动 uvicorn(FastAPI 后端,端口 8000)
3. 启动 rumps 菜单栏 App
4. 启动 pynput 全局快捷键监听
5. 打开 PyWebView 主窗口(默认隐藏,菜单栏点击触发)
6. 等待用户操作

### 5.4 打包产物
- `summary4u.app`(单文件 App,内部含 venv)
- 大小预估:50-80MB(主要是 Whisper 模型外,Python 解释器约 15MB,依赖包约 30MB)
- 模型按需下载,首次使用 small 模型时下载约 460MB

## 6. 风险与权衡 (Trade-offs)

### 6.1 已识别风险
| 风险 | 影响 | 缓解 |
|---|---|---|
| macOS 权限(辅助功能/屏幕录制)申请 | 首次启动体验差 | 快捷键只需"输入监控"权限,提供明确指引 |
| PyInstaller 首次打包慢 | 开发体验差 | 用 `pyinstaller --onedir` 开发版 + `onefile` 发布版 |
| Whisper 模型体积 | App 包体大 | 模型**不打包进 .app**,首次使用按需下载到 `~/Library/Application Support/summary4u/` |
| WKWebView 与 Chrome 行为差异 | 某些 CSS/JS API 不一致 | 主界面功能简单,实测发现个别 API 时改用 webview.expose 桥接 |
| FastAPI 在 App 内 vs 独立运行 | 端口冲突 | App 启动时先检测 8000 端口占用,占用则提示用户关闭 |
| 全局快捷键被其他 App 占用 | 用户改了快捷键仍冲突 | 偏好设置里实时检测冲突,标红提示 |

### 6.2 关键决策对比
- **PyWebView vs Tauri**: Tauri 性能更好(原生 WebView)但需要重写前端为独立 SPA;PyWebView 直接复用现有 WebUI,2 周内可出活
- **rumps vs 自己用 NSStatusBar**: rumps 简单,自己写控制更细但工作量大
- **pynput vs keyboard**: pynput 跨平台,keyboard 性能更好但只在 macOS 稳定;选 pynput 留出未来跨平台余地

### 6.3 不做的选择(明确拒绝)
- ❌ **PyQt/PySide**: 太重,WebUI 直接复用更经济
- ❌ **Electron**: 100MB+ 体积,Python 后端要通信反而麻烦
- ❌ **SwiftUI 重写**: 1 个月+ 工作量,V1 不值得
