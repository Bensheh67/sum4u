## 概述

视频分类器用于根据视频的标题、描述等元数据，自动判断视频类型并选择最合适的总结模板。

**设计目标**：
- 根据视频内容特征自适应选择总结模板
- 支持 4 种核心视频类型
- 提供可解释的分类结果
- 支持混合类型检测

---

## 视频类型定义

| 类型 | 说明 | 适用模板 |
|------|------|----------|
| `learning` | 学习类视频，强调知识框架和概念理解 | 课堂内容 |
| `tutorial` | 教程类视频，强调操作步骤和实战技能 | 短视频素材 |
| `review` | 评测类视频，强调对比和体验 | 精华摘要 |
| `interview` | 访谈类视频，强调观点和对话 | 精炼摘要 |

## 分类原理

### 1. 关键词权重匹配

每种类型定义 2 组关键词规则，权重不同：

```
TYPE_RULES: dict[VideoType, list[dict]] = {
    "tutorial": [
        {"patterns": ["教程", "入门", "从零开始", "how to", ...], "weight": 1.0},
        {"patterns": ["学会", "掌握", "精通", ...], "weight": 0.8},
    ],
    ...
}
```

**计算逻辑**：
1. 遍历所有关键词规则
2. 如果文本中包含某关键词，累加其权重
3. 返回总得分和匹配的关键词列表

### 2. 得分计算

```
total_weight = Σ (匹配到的规则权重)
```

### 3. 置信度计算

```
confidence = min(0.4 + 0.6 * (best_score / 3.0), 1.0)
```

| 得分 | 置信度 | 说明 |
|------|--------|------|
| 3.0 | 1.0 | 满分，强烈匹配 |
| 1.5 | 0.7 | 中等得分 |
| 0 | 0.5 | 无匹配，使用默认 |

### 4. 混合类型检测

当次高分 ≥ 主分 × 60% 时，识别为混合类型：

```python
if second_score >= best_score * 0.6:
    secondary_type = second_type
```

**示例**：
- "深度学习实战教程：从原理到代码实现"
- tutorial 得分: 0.8
- learning 得分: 0.6
- 0.6 ≥ 0.8 × 0.6 = 0.48 → 混合类型

## 局限性

1. **依赖关键词**: 无法理解语义，只能匹配预定义关键词
2. **单语言**: 关键词主要针对中文，英文支持有限
3. **边界情况**: 标题不明显时可能分类不准确
4. **静态规则**: 无法适应新类型，需要手动扩展规则
