## 1. 设计系统继承

### 1.1 完整沿用 DESIGN.md 的 token
- **字体**: Instrument Sans (Display) + Source Sans 3 (Body) + JetBrains Mono (Data)
- **色板**: 主色 `#0D9488` teal,学术极简风
- **间距**: 8px 基准,comfortable 密度
- **圆角**: sm 4px / md 8px / lg 12px
- **动效**: 50-700ms 区间,功能型非装饰型

### 1.2 Mac 平台专属补充
- **窗口材质**: 沿用 macOS 系统质感,WebUI 背景 `rgba(255,255,255,0.85)` 配 `backdrop-filter: blur(20px)`(vibrancy)
- **字体渲染**: WebUI 内显式设置 `-webkit-font-smoothing: antialiased`
- **窗口圆角**: 12px(与 lg token 一致),由 PyWebView 设置
- **窗口阴影**: 默认系统阴影,无自定义

### 1.3 主题策略
- **优先跟随系统**(macOS 浅色/深色)
- 用户可在偏好设置中强制覆盖
- 切换通过 CSS `prefers-color-scheme` media query + `localStorage` 覆盖
- 浅色/深色色值完全沿用 DESIGN.md §Color

## 2. macOS Human Interface Guidelines (HIG) 对齐

### 2.1 必遵循
- ✅ **菜单栏图标尺寸**: 22×22pt(@1x),44×44px(@2x Retina),用 SF Symbols 风格的几何图标
- ✅ **图标风格**: 线性,2px stroke,圆角端点
- ✅ **菜单栏 popover**: 宽度 320px,内边距 16px
- ✅ **系统通知**: 走 macOS Notification Center,不自定义 UI
- ✅ **窗口控制**: 红黄绿三色按钮保留,不做自定义

### 2.2 平台特性
- **支持 Touch Bar**(如有):显示"开始总结"按钮
- **支持 Stage Manager**:主窗口可作为 Stage
- **支持 Handoff**: 不做(V1)
- **支持 Live Activities**: 不做(V1)

### 2.3 不做的(HIG 红线)
- ❌ 替换系统菜单栏/状态栏
- ❌ 自定义窗口控制按钮
- ❌ 修改 dock 菜单(只做基础 dock 拖放)
- ❌ 拦截 Cmd+Q 退出行为(用户能正常退出)

## 3. 组件库 (桌面 App 专属)

### 3.1 菜单栏图标 (Menu Bar Icon)

#### 状态变体
| 状态 | 图标 | 颜色 | 含义 |
|---|---|---|---|
| **Idle** | 圆形外框 + 中心点 | 模板色(自动适配浅深) | 无任务 |
| **Running** | 圆形外框 + 中心脉冲点 | `#0D9488` teal | 有任务运行中 |
| **All Done** | 圆形外框 + 中心实心点 | `#10B981` 绿 | 全部任务完成(5秒后回 Idle) |
| **Error** | 圆形外框 + 中心感叹号 | `#EF4444` 红 | 任务失败 |

#### 设计原则
- **可识别度**: 眯眼看能区分 4 种状态
- **一致性**: 与主窗口 logo 同一视觉语言
- **细节**: 22×22pt 内 4px 安全区(图标外缘 2px 不放内容)
- **动画**: Running 状态的脉冲周期 1.5s,`opacity: 0.3 → 1.0 → 0.3`

#### 图标设计
```
Idle:        Running:       Done:         Error:
  ╭───╮         ╭───╮         ╭───╮         ╭───╮
  │ · │         │ ◉ │         │ ● │         │ ! │
  ╰───╯         ╰───╯         ╰───╯         ╰───╯
```

### 3.2 快速输入浮窗 (Quick Input Popover)

#### 触发
全局快捷键 `⌘⇧S`(可在偏好设置改)→ 浮窗在屏幕中央偏上(避开菜单栏)显示

#### 视觉
```
┌──────────────────────────────────────────┐
│  ✦ 总结音视频                          ✕  │
├──────────────────────────────────────────┤
│                                          │
│  ┌────────────────────────────────────┐  │
│  │ 🔗 输入 YouTube/B站/抖音链接...    │  │ ← 输入框,focus 时高亮 teal
│  │                                    │  │
│  └────────────────────────────────────┘  │
│                                          │
│  模板: [📚 课堂笔记 ▼]                   │ ← 下拉,沿用 WebUI 7 套模板
│  模型: [small ▼]                         │
│                                          │
│              [取消]    [✨ 开始总结]      │ ← Primary 按钮右下角
└──────────────────────────────────────────┘
```

#### 规范
- **尺寸**: 480 × 280px(固定)
- **位置**: 屏幕水平居中,垂直 30% 处(用户视线舒适区)
- **背景**: `rgba(255,255,255,0.95)` + `backdrop-filter: blur(30px)`
- **圆角**: lg (12px)
- **阴影**: `0 8px 32px rgba(0,0,0,0.12)`,`0 2px 8px rgba(0,0,0,0.04)`
- **动画**: 浮窗出现 fade-in 150ms + scale 0.96→1.0 ease-out
- **关闭**: 点 ✕ / Esc / 提交后 200ms 淡出

#### 交互细节
- **输入框**: focus 时自动粘贴剪贴板(若剪贴板是 URL)
- **Enter**: 提交(若输入框非空)
- **Esc**: 关闭
- **失焦自动关闭**: 不做(避免误操作,用户需主动关闭)

## 4. 交互流程图

### 4.1 主流程(从快捷键到结果)
```
用户按 ⌘⇧S
  ↓
浮窗 fade-in (150ms)
  ↓
用户输入 URL + 选模板
  ↓
按 Enter
  ↓
浮窗 fade-out (200ms) + 提交
  ↓
FastAPI 接收任务
  ↓
菜单栏图标变 Running(脉冲)
  ↓
主窗口(若已开)实时更新进度
  ↓
任务完成
  ↓
菜单栏图标变 Done(1.5s) → Idle
系统通知弹窗
  ↓
用户点击通知 → 主窗口定位到该条历史
```

### 4.2 拖放流程
```
用户在 Finder 选中文件
  ↓
拖到 Dock 上的 summary4u 图标
  ↓
PyWebView 接收文件路径
  ↓
判断文件类型
  ├─ .mp3/.wav/.m4a/.mp4/.flac → 转录 + 总结
  └─ .txt/.md → 直接总结
  ↓
任务入队
  ↓
... 同上 4.1
```

### 4.3 错误处理
```
任务失败
  ↓
菜单栏图标变 Error(红)
  ↓
主界面显示错误卡片(可点重试)
  ↓
不弹通知(避免打扰)
  ↓
错误日志写入 ~/Library/Logs/summary4u/error.log
```

## 6. 桌面 App 专属动效

| 场景 | 动效 | 时长 | 缓动 |
|---|---|---|---|
| 浮窗出现 | scale 0.96→1.0 + opacity 0→1 | 150ms | ease-out |
| 浮窗关闭 | scale 1.0→0.96 + opacity 1→0 | 200ms | ease-in |
| 主窗口打开 | macOS 默认 genie 或 scale | 系统默认 | - |
| 进度条更新 | width 过渡 | 200ms | ease-in-out |
| 任务完成 | checkmark draw 200ms + 卡片淡出 | 200ms + 1500ms 停留 | ease-out |
| 菜单栏图标脉冲 | opacity 0.3→1.0→0.3 | 1500ms loop | linear |
| 拖放区 hover | 背景色 + 边框色过渡 | 200ms | ease-out |
| 错误提示 | shake ±4px 3 次 | 300ms total | ease-in-out |

**严格遵守 DESIGN.md §Motion**: 无装饰性动画,全部服务于功能反馈。

## 7. 可访问性 (Accessibility)

### 7.1 必做
- ✅ **键盘可达**: Tab 顺序合理,所有交互有 focus 态
- ✅ **快捷键提示**: 按钮加 `aria-keyshortcuts`
- ✅ **屏幕阅读器**: 进度条用 `role="progressbar"` + `aria-valuenow`
- ✅ **对比度**: 所有文字 ≥ WCAG AA(4.5:1)
- ✅ **动效减弱**: 尊重 `prefers-reduced-motion`,禁用非必要动效

### 7.2 字体缩放
支持 macOS 系统字体大小设置,UI 同步缩放:
- Body 默认 14px
- 系统设"大" → 16px
- 系统设"特大" → 18px
- 布局用 rem,自动响应
